# 单章翻译工作流 (Translate Single Chapter)

## 前置条件
- 章节原文文件已存在于 `章节原文/第X章/第X章原文.md`
- 已阅读 `.agent/skills/translation_sop.md` 中的术语库和规范

## 工作流步骤

### 1. 读取原文
// turbo
- 读取 `章节原文/第X章/第X章原文.md`
- 确认章节编号和标题
- 统计原文段落总数并报告

### 2. 文本预分析
- 识别本章是否包含：
  - 古诗词 → 标记为 `[POEM]` 待人工重构
  - 打戏/动作戏 → 标记需要强化动词
  - 文化梗/字谜 → 标记需要 TN 注释，字谜类（如《断肠谜》）必须标记为 `[PUZZLE]` 并停止 AI 自动翻译
  - 角色首次登场 → 确认术语库中有对应翻译

### 2.5 【强制】术语注入确认（翻译前必须执行）
- 对本章出场的主要角色，逐一执行 `grep_search` 在 `.agent/knowledge/characters/` 中检索确认（至少 3 位）
- 对本章 3 个以上关键文化词，逐一执行 `grep_search` 在 `.agent/knowledge/terminology/` 中检索确认
- **若无法检索到对应词条，必须停止并向用户报告，禁止凭推断自行创造译名**
- 将检索到的术语完整填入 Prompt 模板中的 `{插入术语库}` 占位符，填充完毕后方可继续

> **⚠️ 用户确认点**（等待用户回复后方可进入 Step 3）：
> - 文件路径确认：`章节原文/第X章/第X章原文.md`
> - 原文段落数：XX 段
> - 含特殊内容标记：`[POEM]` X 处 / `[PUZZLE]` X 处 / `[TN]` X 处
> - 术语检索覆盖率：已确认 XX 条
> - **是否继续执行初译？**

### 3. AI 初译 (Draft Translation)
- 按照 `translation_sop.md` 中的 Prompt 模板翻译（术语库已在 2.5 中注入）
- 每次输入不超过 1000 中文字，按场景自然段划分，不在对话中间切断
- 保持段落结构一致，**严禁合并或拆分原文段落**
- `[POEM]` / `[PUZZLE]` 标记段落：输出占位符 `[待人工重构]`，跳过 AI 翻译
- **【段落校验】** 翻译完成后必须报告：`原文段落数 XX → 初译段落数 XX`，若差异 > 0 需逐条说明
- 输出到 `章节原文/第X章/第X章初译_对照.md`（中英双语对照格式）

### 4. L1 双语精校 (Bilingual Post-Editing)
- **【段落校验】** 接收初译稿，核对段落数与 Step 3 报告一致，不符则必须先排查
- 按照 L1 检查清单逐项审核（代词纠错、隐藏主语补全、诗词占位核查、术语一致性）
- **红线**：L1 精校只允许修正错误，不允许改写风格或添加原文不存在的细节
- **【段落校验】** 输出时报告：`输入段落数 XX → 输出段落数 XX`
- 输出到 `章节原文/第X章/第X章L1校审_对照.md`

### 5. L2 母语润色 (Native Polishing)
- 语调校准：检查女主的"爽感"与反差萌是否忠实于原文的字面情绪，不得主动拔高
- 句式重构：拆分超过 40 词的单一长句，加快阅读节奏，但**不得跨句重组段落**
- **对话标签约束**：仅在原文有明确情绪信号时（如原文含"冷笑""轻声""厉声"等词）方可补充对应对话标签（sneered / murmured / said coldly 等）；**严禁主动推断角色情绪并添加副词**
- **红线**：不得更改已核验的术语、人名、诗词方案；不得增加原文不存在的动作或心理描写
- **【段落校验】** 输出时报告：`输入段落数 XX → 输出段落数 XX`
- 输出到 `章节原文/第X章/第X章L2润色_对照.md`

### 6. QA 质检
- **【段落校验】** 核对段落数与 Step 5 报告一致
- 格式标准化检查（斜体、诗词排版、拟声词）
- 术语一致性扫描（最终防线，全篇扫描所有专有名词）
- TN 注释完整性检查（所有 `[TN]` 标记是否已填写内容）
- 输出终稿到 `章节原文/第X章/第X章终稿_对照.md`

### 7. 纯英译稿提取
- 从终稿对照文件中剔除全部中文原文行，仅保留英文翻译行
- 保留 Markdown 格式（斜体、引用块 `>`、`[TN: ...]` 注释）及段落空行
- **严禁将全文直接打印在对话框**，必须使用文件写入工具直接生成文件
- 输出到 `章节原文/第X章/第X章纯英译稿.md`

### 8. 更新项目日志
// turbo
- `"$(Get-Date -Format 'yyyy-MM-dd HH:mm:ss') - 成功完成单章翻译：第X章" | Out-File -FilePath project_log.md -Append -Encoding UTF8`

## 输出文件命名规范

每章目录下最终包含：
```
第X章/
├── 第X章原文.md          # 中文原文
├── 第X章初译_对照.md      # AI初译稿（中英双语对照）
├── 第X章L1校审_对照.md    # L1双语精校稿
├── 第X章L2润色_对照.md    # L2母语润色稿
├── 第X章终稿_对照.md      # QA后终稿（中英对照）
├── 第X章纯英译稿.md       # 纯英文交付稿
└── 第X章术语与人设.md     # 本章术语库与角色档案
```
