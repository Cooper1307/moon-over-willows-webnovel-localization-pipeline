# 批量翻译工作流 (Batch Translation)

## 用法
用户指定章节范围，例如：`/batch_translate 第1章-第10章`

## 前置条件
- 所有目标章节的原文文件已存在于 `章节原文/第X章/第X章原文.md`
- 已阅读 `.agent/skills/translation_sop.md`

## 工作流步骤

### 1. 确认范围
- 解析用户指定的章节范围（起始章-结束章）
- 检查所有原文文件是否存在
- 列出待翻译章节清单

### 2. 流水线模式执行
按照本地化策略中的流水线作业模式：

```
AI初译 第6-10章  →  同时进行
L1精校 第3-5章   →  同时进行  
L2润色 第1-2章   →  同时进行
```

实际操作中（单人+AI模式），按顺序逐章执行 `translate_chapter` 工作流。

### 3. 逐章翻译
对范围内每一章，执行 `/translate_chapter` 工作流：
1. 读取原文
2. 文本预分析
3. AI 初译
4. L1 精校
5. L2 润色
6. QA 质检

### 4. 批量完成后汇总
- 统计已完成章节数
- 列出遇到的特殊问题（诗词、文化梗等）
- 更新翻译进度记录

### 5. 记录日志
// turbo
- `"$(Get-Date -Format 'yyyy-MM-dd HH:mm:ss') - 批量翻译执行完毕，进度已更新" | Out-File -FilePath project_log.md -Append -Encoding UTF8`
