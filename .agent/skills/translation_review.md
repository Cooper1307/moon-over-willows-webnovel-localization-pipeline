# 翻译专业审校技能 (Professional Translation Review)

本技能定义了对指定文件进行深度专业审校的标准化规程。当接收到审校指令时，Agent 必须以极高的精确度执行以下八项质量检查，并严格按照交付标准输出审校件。

## 核心审校清单 (QC Checklist)

请针对需要审核的文件，逐项比对并修正以下翻译质量问题：

1. **漏译排查 (Omission Check)**：逐句核对原文，补全所有缺失的译文段落、术语、脚注及标题（如有附图需补充图表标题）。
2. **过度意译修剪 (Over-translation Control)**：对脱离原意的自由发挥进行严厉标记并改写。必须确保译文在语义、语气、风格上与原文（以及 SOP 设定的人设语调）保持一致。
3. **语境不符修正 (Contextual Alignment)**：结合网文的章节主题、受众定位及前后文逻辑，调整用词、语序与文化参照，使表达符合英语网文或受众目标语境。
4. **逻辑错误修复 (Logical Integrity)**：仔细检查因果、转折、并列等逻辑连接词，修正因语序颠倒或理解偏差导致的推理链断裂。
5. **术语一致性核验 (Terminology Consistency)**：扫描全局 `.agent/knowledge` 库，建立或提取本章的关键术语表，全篇统一专有名词、招式词汇及特定称谓。
6. **语言规范与格式化 (Grammar & Formatting Norms)**：消除错别字、病句、标点误用，并确保对话、内心独白（*斜体*）等特殊格式规范无误。
7. **可读性优化 (Readability Enhancement)**：拆解生硬死板的超长复合句，在保留东方文化严谨度的同时打磨阅读的顺畅感，确保海外小白读者无需反复回读即可理解爽感脉络。

## 强制交付标准 (Deliverable Standards)

每次完成审校后，必须向用户输出以下三项交付物（通常生成对应的 `.md` 报告文件或直接在回话中以清晰区块列出）：

1. **修订痕迹版 (Diff Track Changes)**：使用代码区块（如 ````diff .... ````）保留并高亮显示原始初稿与修改后版本的对读记录。
2. **清洁版 (Clean Final Copy)**：生成一份没有任何校对标记，可以直接用于排版分发的纯净版译文终稿。
3. **审校报告 (Review Report)**：总结本次审校发现的问题，详细阐述：
   - 问题分类总结
   - 改写依据背后的逻辑
   - 剩余翻译风险提示（例如文化梗存在不可译风险、推荐使用注释解决的建议等）。
