## 四、输出格式模板

每章生成一个 `第X章术语与人设.md` 文件，保存到 `章节原文/第X章/` 目录。

严格遵循以下模板结构：

```markdown
# 第X章 术语库 & 角色人设档案

> 章节标题：[从原文第一行提取]

---

## 一、本章术语提取

### [分类名称]（如：人名与称谓 / 官职与机构 / 地点与居所 / 文化特色 / 婚姻法律 等）

| 中文 | 建议英文 | 状态 | 备注 |
|------|----------|------|------|
| XXX | XXX | `[已有]` 或 `[新增]` | 简短说明 |

（可有多个分类表格，按本章实际内容灵活分类）

---

## 二、本章角色人设

### [角色名]

| 维度 | 详情 |
|------|------|
| **身份** | ... |
| **外貌** | ... |
| **性格特征** | ... |
| **语言风格** | ... |
| **英文语调** | ... |
| **代表台词** | ... |
| **翻译要点** | ... |

（可包含多个角色，新登场角色详细记录，已有角色仅记录本章新增/变化信息）

---

## 三、本章翻译挑战

1. **[挑战名称]**：具体描述及建议处理方式
2. ...
```

### 格式规范要点

1. **状态标记**：
   - `[已有]`：该术语在 `.agent/skills/translation_sop.md` 或前序章节中已有记录
   - `[新增]`：本章首次出现的新术语
2. **分类灵活**：根据每章内容实际出现的术语类型灵活分组，不必强求所有章节分类一致
3. **角色档案**：新角色需完整记录，老角色仅记录本章新增的性格/行为/状态变化
4. **翻译挑战**：标记诗词、文化梗、打戏、语调碰撞等需要人工重点处理的段落

1. **逐章阅读原文**：每章原文在 `章节原文/第X章/第X章原文.md`，必须完整阅读后再提取
